1️⃣ Listas por Comprensión Anidadas: Gestión de Inventario

Este código demuestra el uso de listas por comprensión para realizar operaciones de análisis de datos de inventario de manera concisa y eficiente. Las listas por comprensión son una forma elegante y Pythónica de crear nuevas listas a partir de iterables existentes.

🔑 Conceptos Clave

  • Listas por Comprensión: Se utilizan para crear nuevas listas aplicando una expresión a cada elemento de un iterable.
  • Validación de Entrada: Incluye bucles while y bloques try-except para asegurar que el usuario ingrese valores numéricos no negativos.
  • Análisis de Inventario: Las listas por comprensión se usan para extraer nombres, calcular subtotales, filtrar productos y más.

📋 Funcionalidades

  • Ingreso de datos de 3 productos con validación
  • Extracción de nombres de productos
  • Cálculo de subtotales (precio × cantidad)
  • Cálculo del valor total del inventario
  • Filtrado de productos con precio > $100
  • Identificación de productos con stock bajo (cantidad < 10)

💡 Ejemplo de Salida

=== ANÁLISIS DE INVENTARIO ===
1. Datos originales: [['Laptop', 1200, 5], ['Mouse', 25, 50], ['Teclado', 75, 20]]
2. Nombres: ['Laptop', 'Mouse', 'Teclado']
3. Subtotales: [6000, 1250, 1500]
4. Total inventario: $ 8750
5. Productos con precio > $100: ['Laptop']
6. Stock bajo (cantidad < 10): ['Laptop']

2️⃣ Listas como Colas: Sistema de Gestión de Pedidos

Este ejemplo ilustra la implementación de la estructura de datos Cola (Queue) utilizando collections.deque de Python. Las colas funcionan bajo el principio FIFO (First In, First Out), donde el primer elemento en entrar es el primero en salir.

🔑 Conceptos Clave

  • collections.deque: Una lista optimizada para inserciones y eliminaciones rápidas en ambos extremos.
  • Clase SistemaPedidos: Modela un sistema de pedidos de restaurante con métodos para agregar y procesar pedidos.
  • Principio FIFO: Los pedidos se procesan en el orden en que fueron recibidos.

📋 Métodos Principales

  • agregar_pedido(cliente, items): Añade un nuevo pedido al final de la cola
  • procesar_siguiente_pedido(): Elimina y procesa el pedido del principio de la cola
  • mostrar_estado(): Proporciona una visión general del sistema

🍳 Caso de Uso

Simulación de un restaurante donde los pedidos se toman en orden y se procesan en ese mismo orden, garantizando que ningún pedido se pierda y se procesen en orden de llegada.

3️⃣ Listas como Pilas: Implementación de Stack

Este código implementa la estructura de datos Pila (Stack) utilizando una lista de Python, adhiriéndose al principio LIFO (Last In, First Out), donde el último elemento en entrar es el primero en salir.

🔑 Conceptos Clave

  • Clase Pila: Implementa los métodos esenciales de una pila.
  • Operaciones O(1): Las operaciones de apilar y desapilar tienen complejidad temporal constante.
  • Métodos Mágicos: __len__, __str__ y __repr__ mejoran la usabilidad.

📋 Métodos Principales

  • apilar(elemento): Agrega un elemento al tope de la pila
  • desapilar(): Elimina y retorna el elemento del tope
  • ver_tope(): Retorna el elemento del tope sin eliminarlo
  • esta_vacia(): Verifica si la pila está vacía
  • tamano(): Retorna el número de elementos

💡 Aplicación Práctica

El código demuestra cómo usar una pila para invertir una palabra. Por ejemplo, la palabra "DATA" se invierte a "ATAD" usando el principio LIFO: cada letra se apila y luego se desapila en orden inverso.
